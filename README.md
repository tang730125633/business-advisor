# business-advisor · OpenClaw Skill

> 为企业经营者（电力行业 / 可泛化到其他行业）提供基于 24 本经典商业书籍的战略、团队、投资、转型咨询能力。
> OpenClaw skill + 经营管理知识库 + 一键部署脚本。

**版本**：v1.0 · 2026-04-05
**作者**：Tang + Opus (Anthropic Claude)
**适用**：OpenClaw 2026.3.x+

---

## 这是什么

一个打包好的 **OpenClaw Skill**，让 AI 在面对企业经营者（如戴总 / 龙哥这样的电力行业老板）的问题时，能够：

1. **识别情绪优先** — 焦虑 / 愤怒 / 失眠时先共情再建议
2. **精准召回书籍** — 从 24 本经典商业书（德鲁克 / 芒格 / 克里斯坦森 / 波特 / 鲁梅尔特 / 柯林斯 / 科特勒 / 盖洛普 / 达利欧 等）里调用最对症的框架
3. **落地到行业具体场景** — 不空谈原理，结合电力行业（储能 / EPC / 变电站 / 新能源）给建议
4. **反幻觉铁律** — 不编造客户所在城市、员工人数、具体数字等未知事实
5. **克制输出** — 300 字左右，克服小模型爱长篇大论的毛病

## 实测效果

| 问题 | 未启用 skill | 启用 skill 后 | 改进 |
|---|---|---|---|
| 战略类"新能源风口" | 873 字 + "方哥"幻觉 | **411 字** + 无幻觉 + 反问结尾 | **-53%** |
| 情绪类"被客户骂" | 961 字 + 念书分析 | **171 字** + 纯共情 + 让客户选择 | **-82%** |

见 `skills/business-advisor/golden-examples/` 里的 4 个标杆示例。

## 🚀 一键部署

```bash
# 克隆
git clone https://github.com/tang730125633/business-advisor.git
cd business-advisor

# 安装
./install.sh
```

脚本会自动：
1. 备份现有 `~/.openclaw/openclaw.json`
2. 复制 skill 到 `~/.openclaw/skills/business-advisor/`
3. 复制知识库到 `~/.openclaw/workspace/经营管理知识库/`
4. 合并 4 处配置（`searchMode=query` / qmd 挂载点 / mentionPatterns / feishu.requireMention）
5. 触发 config 热重载（**无需重启 gateway，零中断**）
6. 验证 `business-advisor ✓ Ready`

安装后 **3-8 分钟内** qmd 会完成向量索引（首次下载 embedding 模型可能稍慢）。

详细说明 → [INSTALL.md](./INSTALL.md)

## 核心设计思路

### 四层知识架构

```
┌─────────────────────────────────────────┐
│ SOUL.md         ← 身份（Who I Am）      │  每次会话全局加载
│ MEMORY.md       ← 记忆（What I Know）   │  每次会话全局加载
│ Skill (本项目)  ← 流程（How To Do Well） │  按 description 触发
│ qmd RAG         ← 图书馆（What I Can Look Up）│  对话时检索
└─────────────────────────────────────────┘
```

Skill 是**过程性知识** — 不是"AI 是谁"也不是"AI 知道什么"，而是 **"AI 面对商业问题时该怎么工作"**。把规则放在 Skill 层（而不是 SOUL.md）的好处：

- ✅ **按需加载**：商业咨询场景触发，日常闲聊不污染 context
- ✅ **避开 bootstrap 缓存 bug**：skill 机制绕过 OpenClaw workspace 文件缓存
- ✅ **可迭代**：改 SKILL.md 立即生效，不需要重启 gateway
- ✅ **可复用**：复制一份改关键字就能服务其他行业客户

### 六步咨询工作流（SKILL.md 的核心）

```
1. 情绪识别  →  命中情绪信号？→ 先共情，不念书
2. 诊断问题  →  症状 ≠ 瓶颈，找到真正的痛点
3. 召回书籍  →  1-2 本最对症的，不卖弄
4. 应用思维  →  反过来想 / 能力圈 / 以终为始 / 先诊断
5. 落地行业  →  抽象 + 行业场景 = 有用
6. 克制输出  →  ≤300 字 / 反问 / 让客户选
```

## 配置一览（install.sh 会自动处理）

```json
{
  "memory": {
    "qmd": {
      "searchMode": "query",
      "paths": [
        { "path": "~/.openclaw/workspace/经营管理知识库", "name": "business-wisdom-kb", "pattern": "**/*.md" }
      ]
    }
  },
  "messages": {
    "groupChat": {
      "mentionPatterns": [
        "小婷", "婷婷", "@小婷",
        "帮我", "帮忙", "给我分析",
        "你觉得", "你说", "你看", "你分析",
        "怎么办", "怎么看", "咋整", "如何",
        "咱们", "我问你"
      ]
    }
  },
  "channels": {
    "feishu": {
      "requireMention": true
    }
  }
}
```

**效果**：
- 群里有人闲聊 → 小婷沉默
- 说"小婷你觉得..."或"帮我看看" → 小婷自动响应（无需 @）
- 飞书 reply 到小婷之前的消息 → 自动响应

## 如何迭代

SKILL.md 和 golden-examples 都是纯文本，**编辑后立即生效**。建议迭代节奏：

1. **首周**：真实使用，记录 3-5 个不满意的回答
2. **周末**：对照规则找"是哪条规则没执行"还是"缺了示例"
3. **修改**：要么加规则，要么加 golden example
4. **回归**：用 eval harness 跑测试（见 Tang 的 dev 机器 `~/.openclaw/workspace/eval/`）

## 已知限制

- 《授权》（马凯特）仅有 PDF，未蒸馏
- 《穷查理宝典》《营销管理》基于通识知识蒸馏，建议后续补原文
- 300 字长度规则被小模型理解为"参考上限"，实际可能 300-450 字浮动
- 依赖 OpenClaw 2026.3.x+ 的 `mentionPatterns` 配置支持
